# 清洗 Skill V1

## 目标

把“单 PDF -> 知识点图 JSON”拆成稳定的清洗步骤，便于后续从人工版本切换到 API 大模型版本。

## 建议流程

1. 文本切分
   - 先按页切分
   - 再按句子切分
   - 保留页码，便于证据回填

2. 初筛候选概念
   - 从标题、定义句、枚举项、运算名、约束名、模型名中抽词
   - 对中英文术语做初步归并

3. 噪声剔除
   - 删除页眉页脚、目录残片、章节号、表格示例值、人名、编号字段
   - 删除“只有在例子里有意义”的实体

4. 概念归一化
   - 合并同义词、缩写、中英文对应项
   - 生出 `canonical_name`
   - 汇总 `aliases`

5. 定义生成
   - 从定义句或说明句中抽一句作为 `definition`
   - 不足时做最小必要概括

6. 关系抽取
   - 优先抽明确语义关系：
     - `is_a`
     - `part_of`
     - `prerequisite_of`
     - `used_for`
     - `causes`
     - `similar_to`
   - 其次统计共现关系，形成 `CO_OCCURS_WITH`

7. 证据回填
   - 每个节点至少附 1 条 `evidence_refs`
   - 每条证据都要带页码和摘录

8. 后计算重要性
   - 节点和边生成完后再计算 `importance_score`
   - 初版可按中心度或加权度

## 失败模式

- 把“学生、学号、李勇”这类例题实体当作知识点
- 把“2.1.1”“本章主要内容”这种结构噪声当作概念
- 把“关系数据结构及形式化定义”这种章节标题直接塞进图
- `RELATES_TO` 过度泛化，变成所有节点都互相关联
- `CO_OCCURS_WITH` 过密，导致图像毛线球

## 评估要点

- 图里是不是主要都是可教学概念
- 节点定义是不是简洁准确
- 关系类型是不是可解释
- 证据能不能回到页码
- 图看起来有没有明显噪声团
