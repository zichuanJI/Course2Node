# 停用词与筛选规则 V1

## 目标

减少“统计上高频但教学上无价值”的节点。

## 一类噪声：版式与结构噪声

直接丢弃：

- `DBch02`
- `第二章`
- `本章主要内容`
- `2.1`
- `2.1.1`
- `2.3.3`
- `小结`
- `表 2.1`
- `表 2.2`
- 页码数字

## 二类噪声：例题实体

默认丢弃：

- 人名
  - 张清玫
  - 刘逸
  - 李勇
  - 刘晨
  - 王敏
- 示例字段值
  - 信息专业
  - 计算机专业
- 纯案例中的记录值

这些值可以保留在证据里，但不进知识点图。

## 三类噪声：字段型属性名

这类词要谨慎处理：

- 学号
- 课程号
- 专业号
- 姓名
- 性别
- 年龄

规则：

- 如果当前任务是“课程概念图”，默认不进主图
- 如果任务是“数据库建模示例图”，才允许保留

## 四类保留优先级高的概念

优先保留：

- 模型
- 结构
- 运算
- 约束
- 规则
- 模式
- 语言
- 查询方法
- 完整性类别
- 关键术语的中英文表达

## 章节样本中的高优先概念

- 关系模型
- 关系
- 域
- 笛卡尔积
- 元组
- 属性
- 候选码
- 主码
- 外码
- 关系模式
- 关系数据库
- 关系操作
- 选择
- 投影
- 连接
- 除
- 关系完整性
- 实体完整性
- 参照完整性
- 用户定义的完整性
- 关系代数
- 关系演算
- 元组关系演算
- 域关系演算
- SQL

## 当前版本的筛选原则

- 宁可漏掉少量边缘概念，也不要把噪声概念放进主图
- 节点质量优先于节点数量
- 概念主图优先反映“教学结构”，而不是原文覆盖率
